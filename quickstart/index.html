Quick start

Your first Transitrix artefact — a Goals tree rendered from a YAML file — in a few minutes.

Two paths. The fastest uses the onboarding Skill in Claude Code. The second works anywhere — including behind an enterprise GitHub instance — as long as you can reach the VS Code Marketplace and npm.

Either path produces the same result: a .goals.transitrix.yaml file that Transitrix Studio renders as a live diagram as you type.

Path 1 — Onboarding Skill

Requires Claude Code with access to the Claude marketplace. In a Claude Code session:

/plugin marketplace add transitrix/methodology
/plugin install transitrix@transitrix-methodology
/transitrix:onboard

The Skill asks what you want to model first, scaffolds the repo layout (canon/, field/, codex/), and authors a starter Goals tree file. Open it in VS Code with Studio installed — the diagram renders alongside as you type.

Path 2 — Manual walk-through

No Claude Code required. Works without public GitHub access.

  1. Install Transitrix Studio from the VS Code Marketplace.
  2. Open VS Code in a new empty folder.
  3. Create the file views/goals/HELLO.goals.transitrix.yaml.
  4. Paste this into it: 
    notation: goals
spec_version: "0.1"

id: GOALS-HELLO-1
name: "Hello — Goals Tree"
description: "A starter Goals tree."
period: "2026"
version: "0.1"
date: "2026-01-01"
author: Me

goal_types:
  - { name: "Strategic Goal", level: 0 }
  - { name: "Tactical Goal",  level: 1 }

goals:
  - id: GOAL-1
    name: "Expand into new markets"
    type: "Strategic Goal"
    level: 0

  - id: GOAL-1-1
    name: "Enter the German market by Q3"
    type: "Tactical Goal"
    level: 1
    parent: GOAL-1

  - id: GOAL-1-2
    name: "Sign two distribution partnerships"
    type: "Tactical Goal"
    level: 1
    parent: GOAL-1

  - id: GOAL-2
    name: "Improve customer retention"
    type: "Strategic Goal"
    level: 0

  - id: GOAL-2-1
    name: "Reduce churn by 20% in H2"
    type: "Tactical Goal"
    level: 1
    parent: GOAL-2
  5. Studio renders the Goals tree in the preview panel. Edit a goal name — the diagram updates live.

To validate from the command line (requires npm): npx @transitrix/cli validate views/goals/HELLO.goals.transitrix.yaml
