Schnellstart

Ihr erstes Transitrix-Artefakt — ein aus einer YAML-Datei gerenderter Zielbaum — in wenigen Minuten.

Zwei Wege. Der schnellste nutzt den Onboarding-Skill in Claude Code. Der zweite funktioniert überall — auch hinter einem unternehmensinternen GitHub-Server — solange Sie den VS-Code-Marketplace und npm erreichen können.

Beide Wege erzeugen dasselbe Ergebnis: eine .goals.transitrix.yaml-Datei, die Transitrix Studio beim Tippen als Live-Diagramm rendert.

Weg 1 — Onboarding-Skill

Erfordert Claude Code mit Zugriff auf den Claude-Marketplace. In einer Claude-Code-Sitzung:

/plugin marketplace add transitrix/methodology
/plugin install transitrix@transitrix-methodology
/transitrix:onboard

Der Skill fragt, was Sie zuerst modellieren möchten, legt die Repo-Struktur an (canon/, field/, codex/) und erstellt eine erste Zielbaumdatei. Öffnen Sie diese in VS Code mit installiertem Studio — das Diagramm wird beim Tippen daneben gerendert.

Weg 2 — Manuelle Schritt-für-Schritt-Anleitung

Kein Claude Code erforderlich. Funktioniert ohne Zugriff auf öffentliches GitHub.

  1. Installieren Sie Transitrix Studio aus dem VS-Code-Marketplace.
  2. Öffnen Sie VS Code in einem neuen leeren Ordner.
  3. Erstellen Sie die Datei views/goals/HELLO.goals.transitrix.yaml.
  4. Fügen Sie Folgendes ein: 
    notation: goals
spec_version: "0.1"

id: GOALS-HELLO-1
name: "Hello — Goals Tree"
description: "A starter Goals tree."
period: "2026"
version: "0.1"
date: "2026-01-01"
author: Me

goal_types:
  - { name: "Strategic Goal", level: 0 }
  - { name: "Tactical Goal",  level: 1 }

goals:
  - id: GOAL-1
    name: "Expand into new markets"
    type: "Strategic Goal"
    level: 0

  - id: GOAL-1-1
    name: "Enter the German market by Q3"
    type: "Tactical Goal"
    level: 1
    parent: GOAL-1

  - id: GOAL-1-2
    name: "Sign two distribution partnerships"
    type: "Tactical Goal"
    level: 1
    parent: GOAL-1

  - id: GOAL-2
    name: "Improve customer retention"
    type: "Strategic Goal"
    level: 0

  - id: GOAL-2-1
    name: "Reduce churn by 20% in H2"
    type: "Tactical Goal"
    level: 1
    parent: GOAL-2
  5. Studio rendert den Zielbaum im Vorschaufenster. Bearbeiten Sie einen Zielnamen — das Diagramm aktualisiert sich in Echtzeit.

Zur Validierung über die Kommandozeile (erfordert npm): npx @transitrix/cli validate views/goals/HELLO.goals.transitrix.yaml
