Installieren & starten

Zwei Schritte bis zum gerenderten Diagramm: Studio installieren, dann Ihre erste Datei erstellen.

1. Studio installieren

Wählen Sie Ihren Editor — die Erweiterung ist überall dieselbe.

VS Code Cursor / VSCodium / Windsurf JetBrains-IDEs

Oder suchen Sie Transitrix Studio im Erweiterungen-Panel. Sequenz-, Zustands- und ER-/Klassendiagramme rendern nativ über PlantUML — keine zusätzliche Erweiterung nötig. Für leichtere, schnellere Ansichten installieren Sie zusätzlich Markdown Preview Mermaid Support — dasselbe reine Textmodell, kein Lock-in.

2. Repository anlegen — oder nur eine erste Datei

Fügen Sie dies in einen beliebigen Coding-Assistenten ein — Claude Code, Cursor, Copilot Chat, Gemini CLI — im Ordner, in dem Sie arbeiten möchten:

Fetch and follow
https://raw.githubusercontent.com/transitrix/methodology/main/transitrix/skills/onboard/SKILL.md
— for any templates/<file> path it references (incl. ${CLAUDE_SKILL_DIR}/templates/<file>),
fetch it instead from
https://raw.githubusercontent.com/transitrix/methodology/main/transitrix/skills/onboard/templates/<file>

Der Assistent fragt, was Sie modellieren möchten, und legt es entsprechend an:

  • Leerer Ordner — das Transitrix-Repository Ihrer Organisation, von Grund auf: die Zonen canon/, field/, codex/, das transitrix.yaml-Manifest, AGENTS.md-Rollenanleitungen und ein GitHub-Actions-CI-Workflow, der jeden Pull Request validiert.
  • Bestehendes Repository — orientiert sich dort und fügt eine erste Datei hinzu.

In beiden Fällen: Öffnen Sie das Ergebnis in Ihrem Editor — das Diagramm rendert beim Tippen. Skill-Quelle →

Nutzen Sie den Claude-Code-Plugin-Workflow?

Gleiches Ergebnis, über die Terminal-claude-CLI:

/plugin marketplace add transitrix/methodology
/plugin install transitrix@transitrix-methodology
/transitrix:onboard

Kein Coding-Assistent zur Hand? Klonen Sie das Beispiel-Repository und öffnen Sie eine beliebige .transitrix.yaml-Datei unter canon/ — sehen Sie es gerendert, bevor Sie selbst schreiben.

Einem bestehenden Repository Ihres Teams beitreten

Jemand hat das Modell bereits aufgesetzt — klonen Sie es und tragen Sie bei wie bei jeder anderen Änderung:

  1. Repository klonen, im Editor öffnen (Viewer installiert — siehe oben).
  2. AGENTS.md im Stammverzeichnis lesen — Zonen, Notationen, Konventionen.
  3. Datei bearbeiten oder hinzufügen, dann npx @transitrix/cli validate <datei>.
  4. Pull Request öffnen — die CI validiert automatisch erneut.
