Installieren & starten

Installieren Sie Transitrix Studio in Ihrem Editor und erstellen Sie Ihr erstes Diagramm — einen aus reinem Text gerenderten Zielbaum, in wenigen Minuten.

Die Viewer installieren

Transitrix-Diagramme sind reiner Text. Zwei Editor-Erweiterungen rendern sie live beim Tippen — zusammen decken sie jede Ebene eines einzigen Modells ab, vom Vorstand bis zur Entwicklung.

Transitrix Studio — die geschäftsnahen Ebenen

Wählen Sie Ihren Kanal — die Erweiterung ist überall identisch.

  • VS Code — der VS-Code-Marketplace. Oder suchen Sie Transitrix Studio im Erweiterungen-Panel.
  • Cursor, VSCodium, Windsurf — die Open-VSX-Registry. Dieselbe Erweiterung und Kennung; suchen Sie Transitrix Studio im Erweiterungen-Panel.
  • IntelliJ IDEA — der JetBrains-Marketplace. Installiert auch in die anderen JetBrains-IDEs.

Markdown Preview Mermaid Support — die technischen Ebenen

Studio rendert die geschäftsnahen Ebenen; die technischen Ebenen sind standardmäßiges Mermaid. Installieren Sie neben Studio Markdown Preview Mermaid Support (im VS-Code-Marketplace und auf Open VSX), um Sequenz-, Zustands-, ER- und C4-Komponentendiagramme bis hinunter zur Anwendungs- und Technologieebene zu rendern.

Ein Modell, jede Ebene. Studio rendert die geschäftsnahen Ebenen mit eigens entwickelten Notationen, die über generisches Diagrammieren hinausgehen — seine BPMN-, Prozess- und Aktivitätsansichten, keine abgespeckten Mermaid-Kopien. Mermaid rendert dann die technischen Ebenen, auf die Transitrix nicht spezialisiert ist — dasselbe reine Textmodell, jede Ebene, vom Vorstand bis zur Entwicklung. Und nichts bindet Sie: Transitrix baut auf offenen Standards auf (ArchiMate 3.2, BPMN 2.0), die technischen Ansichten sind standardmäßiges Mermaid, das in jedem Werkzeug rendert, und Ihre Dateien bleiben diff-barer Text auf einer offenen, MIT-lizenzierten Methodik — lesbar mit oder ohne unsere Werkzeuge.

Ihr erstes Diagramm

Mit installierten Viewern erstellen Sie Ihr erstes Transitrix-Artefakt: einen Zielbaum, gerendert aus einer .goals.transitrix.yaml-Datei. Zwei Wege.

Der schnelle Weg — Onboarding-Skill

Erfordert Claude Code mit Zugriff auf den Claude-Marketplace. Der Skill legt ein Repository an — oder orientiert Sie in einem, das Ihre Organisation bereits hat — und erstellt Ihre erste Datei. In einer Claude-Code-Sitzung:

/plugin marketplace add transitrix/methodology
/plugin install transitrix@transitrix-methodology
/transitrix:onboard

Der Skill fragt, was Sie zuerst modellieren möchten, legt die Repo-Struktur an (canon/, field/, codex/) und erstellt eine erste Zielbaumdatei. Öffnen Sie diese in VS Code — das Diagramm wird beim Tippen daneben gerendert. Skill auf GitHub ansehen →

Der manuelle Weg

Kein Claude Code erforderlich — und kein Zugriff auf öffentliches GitHub nötig.

  1. Öffnen Sie VS Code in einem neuen leeren Ordner (Studio installiert — siehe oben).
  2. Erstellen Sie die Datei views/goals/HELLO.goals.transitrix.yaml.
  3. Fügen Sie Folgendes ein: 
    notation: goals
spec_version: "0.1"

id: GOALS-HELLO-1
name: "Hello — Goals Tree"
description: "A starter Goals tree."
period: "2026"
version: "0.1"
date: "2026-01-01"
author: Me

goal_types:
  - { name: "Strategic Goal", level: 0 }
  - { name: "Tactical Goal",  level: 1 }

goals:
  - id: GOAL-1
    name: "Expand into new markets"
    type: "Strategic Goal"
    level: 0

  - id: GOAL-1-1
    name: "Enter the German market by Q3"
    type: "Tactical Goal"
    level: 1
    parent: GOAL-1

  - id: GOAL-1-2
    name: "Sign two distribution partnerships"
    type: "Tactical Goal"
    level: 1
    parent: GOAL-1

  - id: GOAL-2
    name: "Improve customer retention"
    type: "Strategic Goal"
    level: 0

  - id: GOAL-2-1
    name: "Reduce churn by 20% in H2"
    type: "Tactical Goal"
    level: 1
    parent: GOAL-2
  4. Studio rendert den Zielbaum im Vorschaufenster. Bearbeiten Sie einen Zielnamen — das Diagramm aktualisiert sich in Echtzeit.

Zur Validierung über die Kommandozeile (erfordert npm): npx @transitrix/cli validate views/goals/HELLO.goals.transitrix.yaml

Einem bestehenden Repository Ihrer Organisation beitreten

Jemand in Ihrem Team hat das Modell bereits aufgesetzt. Sie legen nichts an — Sie klonen es, sehen es rendern und machen Ihre erste Änderung wie jeden anderen Beitrag.

  1. Klonen Sie das Repository und öffnen Sie es in VS Code (beide Viewer installiert — siehe oben): 
    git clone <repo-url-ihrer-organisation>
  2. Öffnen Sie eine beliebige .transitrix.yaml-Datei unter canon/ oder views/. Studio rendert das echte Diagramm Ihrer Organisation daneben.
  3. Lesen Sie AGENTS.md im Repository-Stammverzeichnis — der Orientierungsleitfaden, den das Repo mitbringt: wo jede Zone liegt (canon/, field/, codex/), welche Notationen verwendet werden und die lokalen Konventionen.
  4. Nehmen Sie eine Änderung vor — ein Element bearbeiten oder hinzufügen — und validieren Sie sie: 
    npx @transitrix/cli validate <datei>
  5. Öffnen Sie einen Pull Request. Architekturänderungen werden wie Code geprüft, und die CI des Repositorys führt die Validierung bei jedem PR erneut aus.

Das ist die ganze Schleife — klonen, rendern, ändern, validieren, PR — derselbe Review-as-Code-Workflow, den Ihr Team bereits für Quellcode nutzt.
